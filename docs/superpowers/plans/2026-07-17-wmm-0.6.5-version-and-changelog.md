# WMM 0.6.5 Version and Changelog Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Advance all active source version surfaces to 0.6.5, add a concise five-language feature changelog, and preserve the published 0.6.0 update manifest.

**Architecture:** Keep `APP_VERSION` and package metadata as the source-version contract while treating `packaging/update-manifest.json` as immutable published-artifact metadata. Prepend a localized 0.6.5 entry to the existing changelog structure and protect the requested writing rule with a repository test.

**Tech Stack:** Python 3.11, unittest, Vue metadata, Windows version resources, Markdown.

## Global Constraints

- Source version is exactly `0.6.5`.
- Published update manifest remains exactly `0.6.0`.
- Changelog text describes only user-visible functions and fixes.
- All five built-in languages remain synchronized.
- Do not package, publish, commit, tag, or create a release.
