# WMM 0.6 Complete Implementation Plan

**Goal:** Implement the approved 0.6 source changes: save-game MOD actions, scoped Ctrl+A, runtime low-consumption mode, startup optimization, live MOD detection, background Workshop refresh, safe file deletion, unsubscribe cleanup, official `.twmods` import, copy-path context action, and synchronized version/changelog updates.

**Constraints:** Preserve the existing dirty worktree, merge with the in-progress game-data feature, modify only repository source/tests/docs, and do not package, publish, commit, tag, or write to the release directory.

## Task 1: Establish focused failing tests

- Extend save-game, scanner/API/storage, settings, launcher/runtime, main, and version tests.
- Extend store, menu, list, settings, save modal, and import UI tests.
- Add focused tests for watcher debouncing, background-job coalescing, deletion-path validation, recycle-bin dispatch, unsubscribe directory cleanup, and `.twmods` parsing.

## Task 2: Implement backend domain services

- Add official profile parser/import service.
- Add safe MOD file operation service.
- Add event-driven MOD watcher and serialized reconciliation coordinator.
- Add bounded Workshop background coordinator.
- Add Windows-native game process detection and runtime lifecycle coordinator primitives.

## Task 3: Refactor API and scanning contracts

- Split cache-only local reconciliation from online metadata refresh.
- Register RPCs for save MODs, deletion preview/execution, official-profile selection/preview/import, background refresh status, live detection, and runtime lifecycle.
- Preserve atomic asset snapshots, revision ordering, playset missing identities, and existing game-data behavior.

## Task 4: Implement frontend state and interactions

- Apply revisioned background scan results without blocking the UI.
- Add scoped Ctrl+A and save-game enable/compare behavior.
- Add copy-path, batch delete preview, dual-source labels, and unsubscribe cleanup flows.
- Add official-profile preview with new/replace playset options and missing-subscription handling.
- Add live-detection setting and background task indicators.

## Task 5: Implement desktop lifecycle

- Add activation event for second-launch focus.
- Route startup directly to idle page when the game is already running.
- Add suspend handshake, static idle page, forced-awake mode, restore flow, and owned-process working-set trimming.
- Stop watchers/jobs in idle mode and reconcile once on resume.

## Task 6: Synchronize 0.6 metadata and five-language copy

- Advance source version references to 0.6.0 while retaining the formal 0.5.0 update manifest.
- Add the 0.6 changelog in all five languages.
- Add every new UI string in all five languages and update documentation.

## Task 7: Verify

- Run focused tests throughout implementation.
- Run the complete Python suite, complete Vitest suite, production frontend build, available lint checks, and `git diff --check`.
- Benchmark local/cached scan paths and inspect changed files for accidental release or unrelated modifications.
