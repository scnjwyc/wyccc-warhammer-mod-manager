# WMM 0.8.1 播放集隔离与移除友伤设置修复设计

## 目标

将版本提升至 0.8.1，并修复两个用户可见问题：游戏数据修改面板应重新优先显示“移除单位友伤”和“移除法术友伤”设置；播放集必须按所选游戏隔离，战锤 3 的播放集不能出现在三国中。

## 范围与约束

- 仅修改 `G:\git\wyccc-warhammer-mod-manager` 内的源码、测试和文档。
- 不运行打包流程、不生成 EXE、不更新带有未构建产物哈希的发布清单，也不创建 Git 提交、Tag 或 Release。
- 更新源码版本、Python/Node 元数据、Windows 资源版本、README、全部内置语言更新日志和版本一致性测试至 `0.8.1`。
- 保留现有游戏数据设置、Workshop 订阅校验和启动时补丁生成机制；本次不更改友伤效果的实际游戏逻辑。
- 保留既有用户播放集和排序。所有 0.8.0 以前的无游戏归属播放集迁入 `warhammer3`，不复制到三国。

## 友伤设置的界面修复

0.8.0 新增“单位容量”卡片后，友伤卡片被排在更靠后的内容区域。现有开关、持久化字段和补丁生成逻辑仍然存在，因此修复限定为恢复信息架构，而不是重写功能。

`GameDataModificationModal.vue` 中的卡片顺序改为：

1. 单位规模；
2. 移除单位/法术友伤；
3. 单位容量。

友伤卡片继续使用已有的订阅状态：未订阅 `Dynamic No Friendly Fire` 时开关保持可见但禁用，并显示既有的订阅提示。该顺序确保用户打开面板时先看到此前被后续卡片挤到下方的设置。

## 按游戏隔离播放集

### 数据模型

`playsets` 增加持久化的 `game_id`。数据库迁移将现有行标记为 `warhammer3`，并将旧的全局当前播放集指针写入战锤 3 的游戏专属指针。

每个支持的游戏拥有：

- 一个不可重命名、不可删除的默认播放集；
- 一个独立的当前播放集指针；
- 仅包含自身 `game_id` 的播放集列表；
- 在该游戏内独立校验的播放集名称。

为兼容已有数据库，战锤 3 的默认播放集继续使用原有 `default` ID。三国使用独立的默认 ID。默认播放集由 `is_default` 标记，而不是让前端依赖单一固定 ID。

数据库迁移需要把旧的全局播放集名称唯一性替换为游戏范围内的大小写不敏感唯一性，因此两款游戏可以各自拥有同名播放集。迁移必须保留 `playset_items`、创建时间和更新时间。

### API 与运行时行为

后端从当前 `selected_game` 解析活动游戏 ID，并将其显式传给所有播放集读写操作。`bootstrap`、扫描、创建、导入、切换、重命名、更新和删除都只返回或操作活动游戏的播放集。

若客户端传入另一款游戏的播放集 ID，后端拒绝该请求，不会切换、修改或删除跨游戏数据。切换 `selected_game` 后，下一次扫描/启动载入该游戏独立的当前播放集和 MOD 顺序；切回原游戏则恢复其原先的当前播放集。
