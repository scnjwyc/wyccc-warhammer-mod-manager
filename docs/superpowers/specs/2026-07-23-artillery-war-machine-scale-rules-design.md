# 炮兵与战争机器单位规模规则设计

## 目标

在现有全局单位规模倍率和单体单位规则之外，增加两套互相独立的规则：

- 炮兵单位调整规则
- 战争机器调整规则

两套规则均提供“仅血量”“50%规模”“100%规模”三个选项，并确保单位总血量仍按全局单位规模倍率增长。

## 设置

新增以下设置字段：

- `artillery_unit_mode`
- `war_machine_unit_mode`

合法值：

- `health`：仅提高血量，不增加模型数量。
- `half`：只应用全局倍率增量的 50% 到模型数量。
- `full`：模型数量使用完整全局倍率。

两个字段的默认值均为 `full`。旧设置中不存在字段或字段值无效时，也回退到 `full`，以保持现有行为。

设置需要接入默认设置、持久化、RPC 参数白名单、补丁设置规范化、补丁状态指纹、启动选项摘要和中英文界面文本。

## 单位分类和优先级

分类以 `land_units.category` 为主，同时结合 `main_units.caste` 和现有单体单位字段。每条 `main_units` 记录只解析一次分类，优先级固定为：

1. 人物角色
2. 单体单位
3. 炮兵
4. 战争机器
5. 普通单位

具体规则：

- `main_units.caste` 为 `lord` 或 `hero` 时，归入人物角色规则。
- 非人物角色且 `main_units.num_men = 1`、`main_units.is_monstrous = true` 时，归入现有单体单位规则。
- 单体单位判定不再排除带有 `engine` 或 `num_engines` 的单位。即使其 `land_units.category` 是 `artillery` 或 `war_machine`，也优先归入单体单位规则。
- 其余单位在 `land_units.category = artillery` 时归入炮兵规则。
- 其余单位在 `land_units.category = war_machine` 时归入战争机器规则。
- 其他单位沿用普通单位规则。

人物角色继续使用现有领主与英雄血量设置；单体单位继续使用现有单体“仅血量/规模”设置。

如果多条 `main_units` 记录引用同一条 `land_units` 记录，共享记录采用所有引用者中的最高优先级策略，避免同一条土地单位记录被不同分支重复修改。

## 规模计算

设全局单位规模倍率为 `M`，目标规模倍率为 `S`：

- 仅血量：`S = 1`
- 50%规模：`S = 1 + (M - 1) × 0.5`
- 100%规模：`S = M`

例如全局倍率为 2 时，三种模式的目标规模倍率分别为 1、1.5、2。

规模倍率应用到：

- `main_units.num_men`
- `land_units.num_engines`
- `land_units.rank_depth`
- 非引擎单位的 `land_units.num_mounts`

引擎单位的 `land_units.num_mounts` 延续现有行为，保持不变。

所有可能产生小数的非负模型数量使用标准四舍五入，0.5 向上取整。例如 4.2 取 4，4.5 取 5。`main_units.num_men` 最低为 1；所有结果限制在 I32 范围内。

## 血量补偿

设取整后的实际模型数量比例为 `A = 新 num_men ÷ 原 num_men`。

- 100%规模不进行额外血量补偿，模型数量本身提供完整总血量增长。
- 仅血量时 `A = 1`，单模型血量倍率为 `M`。
- 50%规模时，单模型血量倍率为 `M ÷ A`，使取整后的单位总血量仍尽量精确等于原单位的 `M` 倍。

例如原单位有 3 个模型、全局倍率为 2：

- 50%规模理论数量为 4.5。
- 标准四舍五入后得到 5。
- 实际模型数量比例 `A = 5 ÷ 3`。
- 单模型血量倍率为 `2 ÷ (5 ÷ 3) = 1.2`。
- 最终总血量为原单位的 2 倍。

血量补偿通过 `land_units.bonus_hit_points` 实现，不修改可能被多个单位共享的 `battle_entities` 记录。补偿值使用标准四舍五入并限制在 I32 范围内。

如果需要血量补偿但无法解析所需的 `land_units` 或 `battle_entities` 引用，补丁生成应返回包含单位键和缺失引用的明确错误，不写出部分有效的补丁。

## 补丁生命周期

新增设置必须进入补丁状态指纹，并提升补丁构建器版本。以下变化均需要重新生成补丁：

- 炮兵规则变化
- 战争机器规则变化
- 全局单位规模倍率变化
- 任何参与计算的 Pack 文件修改时间变化

如果全局倍率恢复为 1 倍，且其他游戏数据修改也全部为 1 倍或关闭，则删除现有游戏数据补丁。

## 界面

在现有“单体单位调整规则”附近增加两行设置：

- 炮兵单位调整规则
- 战争机器调整规则

每行使用三个互斥按钮：

- 仅血量
- 50%规模
- 100%规模

两个类别的按钮状态互相独立，并与保存后的实际设置保持一致。

## 验证

后端测试至少覆盖：

- 人物角色优先于战争机器。
- 假想的带引擎单体怪兽优先于炮兵或战争机器。
- 炮兵和战争机器规则互相独立。
- 仅血量模式不修改规模字段并按全局倍率提高血量。
- 50%规模模式使用增量一半的倍率。
- 3 个模型在 2 倍全局倍率下变为 5，并通过血量补偿保持 2 倍总血量。
- 标准四舍五入与现有向上取整行为分离。
- 100%规模保持现有完整缩放行为。
- 引擎单位的 `num_mounts` 保持不变。
- 共享 `land_units` 记录采用最高优先级策略。
- 新设置的默认值、规范化、RPC 传递、持久化、补丁指纹和启动选项摘要。
- 所有修改恢复为关闭状态后删除补丁。

前端验证至少覆盖：

- 两行新设置正确显示。
- 两个类别可以选择不同模式。
- 保存与重新打开后状态保持一致。
- 中英文文本存在且前端构建通过。
