# WMM 0.6 运行生命周期、实时同步、文件操作与官方配置导入设计

**日期：** 2026-07-16
**状态：** 已确认，按推荐方案 A 实施
**目标版本：** 0.6.0（仅更新源码、测试和文档，不执行发布打包）

## 范围

本设计补充已确认的“存档模组操作与 Ctrl+A 全选”设计，并把以下能力纳入 0.6.0：

1. 右键复制 MOD 文件路径。
2. 游戏运行时进入 RimCrow 风格的完整低消耗模式。
3. 移除启动关键路径中的在线工坊请求，缩短可交互时间。
4. 第二次启动 WMM 时唤醒已有窗口，不再只提示“程序已运行”。
5. 默认开启、可关闭的实时 MOD 增删检测。
6. 工坊信息刷新在受限后台线程中运行，不阻塞界面。
7. 导入官方启动器 `modprofiles/*.twmods` 配置。
8. 右键批量删除 MOD 文件并移入 Windows 回收站。
9. 取消订阅后同时清理对应工坊目录并立即更新 MOD 列表。

## 已验证的现状

- 303 个 MOD 的纯本地扫描中位数约为 0.40 秒，读取当前 2 MB 工坊缓存后约为 0.45 秒。
- 当前首次启动约 5 秒的主要阻塞来自同步 Steamworks 依赖查询；PyInstaller 单文件展开约另占 1 秒。
- pywebview RPC 本身已在独立 Python 线程中调用；“刷新工坊信息”阻塞交互是前端使用全局 busy 所致。
- “WMM 已经在运行”只由 `Local\\WycccModManager` 互斥锁触发，与 `Warhammer3.exe` 无关。已观察到 WMM 窗口仍在全屏游戏后方。
- 当前完整界面空闲 CPU 很低，但 WMM 与所属 WebView2 进程树工作集较大；低消耗模式重点是停止后台活动和压缩工作集。

## 1. 运行生命周期与单实例激活

新增统一的运行协调器，状态为：

`normal -> suspending -> idle -> forced-awake -> resuming`

### 游戏检测

- Windows 使用原生进程快照检测 `Warhammer3.exe`，不再周期性创建 `tasklist` 子进程。
- 启动时立即检测一次；正常界面和低消耗页面均由后端每 5 秒复查。
- 游戏状态变化通过单一协调器广播，避免前端和多个服务各自轮询。

### 进入低消耗模式

- 若 WMM 启动时游戏已经运行，窗口直接加载静态低消耗页面，不启动 Vue 主界面、MOD 扫描、缩略图、目录监听或工坊请求。
- 若游戏在 WMM 运行期间启动，先通知前端落盘播放集变更；确认或超时后停止监听与后台任务并加载静态页面。
- 静态页面只保留游戏状态、临时唤醒和退出操作。
- 页面稳定后执行 Python 垃圾回收，并对 WMM 及其拥有的 WebView2 子进程调用 Windows 工作集压缩。
- 游戏运行期间不启动实时检测、在线刷新或缩略图任务。

### 临时唤醒与恢复

- “临时唤醒管理器”重新加载主界面并设置 `forced-awake`。
- 临时唤醒期间仍暂停实时检测和在线后台任务；删除文件和取消订阅在前后端同时禁用。
- 顶部提供“返回低消耗模式”。除非用户点击该按钮或游戏退出，否则不自动再次休眠。
- 游戏退出后自动恢复主界面，执行一次缓存本地扫描，再启动实时监听及允许的后台任务。

### 单实例

- 保留命名互斥锁作为唯一实例判定。
- 新增命名激活事件。第二个进程检测到互斥锁后，通知已有实例恢复、显示并置前窗口，然后退出。
- 如果 Windows 拒绝焦点切换，则闪烁任务栏图标；只有激活通道失败时才显示错误提示。
- 低消耗页面收到激活事件时只置前页面，不自动退出低消耗模式。

## 2. 启动优化

- 初始扫描使用 `cache-only` 模式：读取本地 Pack 和已有元数据缓存，但不调用 HTTP 或 Steamworks。
- `ensure_dependencies` 从普通扫描路径移出，归入后台工坊任务。
- 首个本地列表显示后才根据设置调度在线工坊刷新。
- 保持当前单文件发行形式；本机目标为 2 秒内出现可操作列表。
- 增加阶段计时日志：进程入口、API 初始化、窗口创建、桥接就绪、本地扫描完成、首屏可用和后台刷新开始/结束。

## 3. 实时 MOD 检测

新增 `live_mod_detection` 设置，默认 `true`，设置保存后立即启停监听器。

- 使用基于 Windows `ReadDirectoryChangesW` 的事件监听库，监听游戏 `data` 根目录和工坊根目录。
- 只关注 `.pack` 的创建、删除、修改、移动，以及工坊数字目录的创建、删除和移动。
- 连续事件在最后一次变化后等待约 1 秒；新增或修改文件必须连续两次保持大小和时间戳稳定。
- 每批事件只提交一次 `cache-only` 扫描。扫描过程中有新事件时只追加一次待校准标记。
- 扫描结果携带递增 revision；前端拒绝比当前 revision 更旧的结果。
- 监听溢出或异常时执行一次完整本地校准并重启监听；失败时保留手动重新扫描能力并提示警告。
- 应用自身的复制、删除和取消订阅事件与监听事件合并，避免重复扫描和重复 toast。
- 游戏进入低消耗模式时停止监听；退出后以一次完整本地校准补齐期间变化。

性能目标：监听空闲 CPU 不高于 0.2%，无周期性目录树遍历；文件稳定后 2 秒内更新列表。

## 4. 后台工坊刷新

- 新增受限后台任务协调器，同一时间只运行一个工坊刷新；重复请求返回现有任务状态。
- 前端按钮调用后台刷新动作，不设置全局 busy，仅显示独立状态指示。
- 任务从当前 MOD 快照收集 Workshop ID，分阶段刷新英文详情、本地化详情、依赖和作者信息，再原子写入缓存。
- HTTP 批次和作者请求使用有界线程池；Steamworks 调用保持单批次，禁止无界并发。
- 完成后调度一次 `cache-only` 扫描并原子应用新元数据。
- 失败时保留旧缓存和当前列表；错误通过任务状态和 toast 呈现。
- 游戏启动时请求取消；已经进入的单次外部调用结束后不再执行后续阶段。

## 5. 受控文件删除与取消订阅

### 复制路径

- 右键菜单新增“复制 MOD 路径”。
- 单选复制当前主路径；多选按当前选择顺序复制去重后的主路径，以换行分隔。
- 双来源 MOD 的主路径是 DATA 副本；工坊位置仍由现有“打开工坊文件夹”操作访问。

### 删除 MOD 文件

- 支持多选，并使用两阶段操作：后端先生成预览，前端显示 DATA/工坊文件数量及完整路径，用户确认后再执行。
- 所有目标必须是当前扫描快照中的 `.pack`，且位于已配置的 `data` 根目录或工坊数字目录中；执行前再次校验路径、来源、大小和修改时间。
- 文件统一移入 Windows 回收站，不做永久删除；符号链接只移动链接本身。
- 双来源 MOD 显示“从 DATA 中删除”，且只处理 DATA 副本。扫描后该行转为工坊来源。
- 工坊独占 MOD 的“删除 MOD 文件”只移动所选 `.pack`，保留 Steam 订阅；Steam 以后可能重新下载。
- 批量目标按规范化路径去重。游戏运行时前端禁用，后端也强制拒绝。
- 删除成功后清理对应 DATA 同步记录并执行一次本地校准；播放集中的缺失身份按现有缺失项机制保留。

### 取消订阅

- 批量取消订阅按 Workshop ID 去重；一个工坊项目包含多个 Pack 时只发送一次 Steam 请求。
- Steam 接受取消订阅后，永久删除严格校验后的 `workshop_root/<Workshop ID>` 整个目录。
- 工坊独占 Pack 从列表消失；有 DATA 副本的 Pack 转为 DATA 来源并继续保留。
- 成功和失败按 Workshop ID 分别汇总，不因一个项目失败而跳过其他项目；本地删除失败必须报告完整路径和部分完成状态。
- 游戏运行时前端禁用，后端也强制拒绝，避免被占用文件产生半完成状态。
- 操作结束后即使实时检测关闭，也显式执行一次本地校准。

## 6. 官方启动器配置导入

- 文件选择器默认打开当前用户的 `%APPDATA%\\The Creative Assembly\\Warhammer3\\modprofiles`，过滤 `*.twmods`，不存在时回退到其最近存在的父目录。
- 解析 `mod_lookup_key://...@steam_workshop:1142710/<Workshop ID>@.../<Pack name>`；忽略注释、空行、哈希和时间戳。
- 保留文件行顺序，按 Workshop ID 与 Pack 名双重匹配；重复项只保留第一次。
- 导入前显示预览：已安装、缺失/未订阅、无法识别和最终顺序。
- 预览提供“新建播放集”和“覆盖当前播放集”；默认新建并以文件名作为名称，重名沿用播放集自动加序号规则。
- 缺失工坊项目复用分享码流程：确认后批量订阅，使用 pending 身份保留原位置；实时检测在下载完成后自动解析为实际 MOD。
- 用户拒绝订阅时仍保留缺失占位，不打乱其他 MOD 顺序。

## 7. 与既有 0.6 功能的组合

- 存档“启用模组”“比较模组”和 Ctrl+A 按已确认的独立设计实现。
- Ctrl+A 继续只选择当前聚焦列表接收到的可见结果，自动适配实时扫描后的列表。
- 实时校准应用结果时保留仍存在的选择、当前播放集顺序、搜索条件和排序条件。
- 所有新增文案同步简体中文、英语、韩语、俄语和日语。

## 8. 错误处理与安全边界

- 文件操作不接受前端任意路径，只接受 MOD ID 和不可伪造的预览 token；后端从当前快照重新解析目标。
- 路径比较使用规范化绝对路径和允许根目录，禁止父目录穿越、非数字工坊目录和非 `.pack` 目标。
- 取消订阅只删除与已确认 Workshop ID 完全一致的一级目录。
- 外部任务、监听器和窗口回调在应用退出时有序停止，不遗留非守护线程。
- 本次不写入发行目录，不运行发布打包，不创建 Git 提交、Tag 或 Release。
