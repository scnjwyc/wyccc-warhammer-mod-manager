# 类型顺序、类型排序与界面可读性设计

## 目标

为 MOD 管理器增加可持久化的类型顺序管理、按类型显示排序、右键菜单手动输入类型，并提高截图所示区域的文字可读性。

## 范围

### 类型顺序

- 内置类型与自定义类型统一纳入一个全局顺序。
- 使用现有 `app_state` 持久化类型 ID 顺序，避免改变 `custom_mod_types` 表结构。
- 首次读取时沿用当前默认类型顺序；发现新类型时追加到末尾，删除类型时从顺序记录移除。
- 类型管理窗口为每行提供上移、下移操作，首行和末行对应按钮禁用。
- 移动操作立即调用后端保存，并刷新类型列表；移动期间沿用现有 busy 状态防止重复提交。

### 按类型排序

- 在现有显示排序菜单新增“类型”选项。
- 按持久化类型顺序升序排列；一个 MOD 有多个类型时使用其第一个已分配类型作为排序键。
- 没有有效类型的 MOD 使用“未知”类型；同类型项使用现有文件名稳定排序作为次级键。
- 支持现有升序/降序切换。
- 只要选择了非“优先级”排序，显示列表就使用视觉排序结果，不被手动加载顺序覆盖；“优先级”仍保持现有实际加载顺序行为。

### 右键菜单手动输入

- 在“修改类型”子菜单顶部增加“手动输入”操作，并显示 `Shift + F`。
- 触发后弹出输入框。
- 输入名称与已有类型不区分大小写匹配时，复用已有类型；不存在时创建自定义类型，然后将该类型应用到当前右键选择的一个或多个 MOD。
- 沿用现有批量类型切换逻辑和错误提示；取消输入不产生修改。

### 字体可读性

采用局部可读性调整，不修改全局页面缩放，以避免现有固定面板和按钮布局溢出：

- MOD 详情的 Workshop ID、作者、创建时间、更新时间字号和行高提高。
- 底部继续游戏、启动游戏按钮及播放标记字号提高。
- 类型管理的说明、名称、输入框、默认标签与操作按钮字号提高。
- 设置页 TAB 的主标题、副标题和标记字号提高。
- 对同步放大的紧凑区域补充行高或最小高度，保持文本不重叠。

## 数据流与接口

后端新增类型顺序读取和保存接口，前端 store 负责调用并更新 `modTypes`。排序函数接收类型顺序映射，保持纯函数形式，便于单元测试。TypeManagerModal 仅负责展示和发出移动事件，App/store 负责持久化。ModContextMenu 仅发出手动输入动作，App 负责提示、创建和批量应用。

## 错误处理

- 后端校验顺序列表中的 ID，只保留当前有效类型并追加遗漏类型；未知 ID 不写入数据库。
- 类型创建或排序保存失败时沿用 store 的 busy/toast 错误处理，不改变本地顺序的最终状态。
- 手动输入为空或取消时直接返回；名称冲突按现有类型名称校验处理。

## 测试策略

- 后端 storage/API：验证默认顺序迁移、类型顺序保存、移动后读取、新建追加、删除清理和非法 ID 过滤。
- 前端 store/modSearch：验证类型排序、降序、多个类型和自定义顺序下的稳定次级排序，并验证非优先级排序不受 inactive 自定义顺序覆盖。
- TypeManagerModal：验证所有类型的上下移动事件和边界禁用状态。
- ModContextMenu/App：验证手动输入入口、快捷键标签、已有类型复用、新类型创建及批量应用。
- 完成后运行前端全量测试、后端测试、生产构建和 `git diff --check`。
