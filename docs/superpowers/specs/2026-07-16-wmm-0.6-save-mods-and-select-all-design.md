# WMM 0.6 存档模组操作与 Ctrl+A 全选设计

**日期：** 2026-07-16
**状态：** 已确认
**目标版本：** 0.6.0（源码版本推进，不执行正式打包或发布）

## 目标

为 Wyccc's Mod Manager 增加以下能力：

1. 存档列表中的“启用存档中使用的模组”。
2. 存档列表中的“比较存档中的模组”。
3. 在当前模组列表中按 `Ctrl+A` 全选；存在搜索条件时只选择该列表内的搜索结果。
4. 将应用源码版本推进到 `0.6.0`，并同步源码、测试、文档、Windows 版本信息和五语更新日志。

## 已确认的行为

### 启用存档中的模组

- 行为与参考项目 WH3-Mod-Manager 一致：先替换当前全部启用项，再启用存档中列出的本机已安装模组。
- 启用顺序遵循存档中 Pack 名称的出现顺序。
- Pack 名称匹配不区分大小写。
- 重复 Pack 名称只处理一次，保留第一次出现的位置。
- 本机缺失的模组不自动订阅或下载，只在结果提示及比较窗口中显示。
- 原版 Pack 不加入启用列表，也不作为缺失模组报告。
- 如果存档成功解析且确实不包含模组，当前启用列表会被清空。
- 如果存档读取或解析过程报错，当前启用列表保持不变。
- 替换后的顺序通过现有播放集写入队列即时保存，并沿用现有加载顺序脏状态和依赖警告刷新逻辑。

### 比较存档中的模组

比较结果包含三组，均按 Pack 名称进行大小写不敏感比较：

1. 存档中存在、当前未启用。
2. 当前已启用、存档中不存在。
3. 存档中存在且当前已启用。

原版 Pack 在比较前过滤。已安装模组显示有效名称和 Pack 名称；本机缺失项至少显示存档中的原始 Pack 名称。比较操作是只读操作，不修改播放集。

### Ctrl+A 全选

- `Ctrl+A` 只作用于当前获得焦点的“未启用 MOD”或“已启用 MOD”列表。
- 选择范围使用该列表组件当前收到的 `mods`，因此自动遵循搜索、隐藏项可见性和列表归属。
- 搜索期间只选择当前列表内的搜索结果，不选择被搜索条件过滤掉的模组。
- 两个列表不会被一次快捷键同时全选。
- 空列表上的 `Ctrl+A` 不修改现有选择。
- 输入框、文本域、选择框和可编辑元素保留系统默认全选行为。
- 全选结果按当前可见顺序写入 `selectedIds`；首个可见模组成为 `selectedId` 和选择锚点，以保持详情面板和后续 Shift 选择行为确定。

## 方案选择

采用方案 A：后端按需读取存档，前端复用现有 Pinia 状态完成匹配、启用和比较。

未采用的方案：

- 打开列表时预解析全部存档：会拖慢列表打开速度，并引入缓存失效问题。
- 后端同时维护当前启用状态并直接生成全部 UI 结果：会与 Pinia 形成重复状态源。

## 架构与组件职责

### 后端存档服务

`backend/save_games.py` 继续作为存档目录与文件访问的唯一入口。

新增 `SaveGameService.pack_names(save_name)`：

- 首先调用现有 `require()`，继续执行文件名验证、大小写匹配和路径穿越防护。
- 按参考实现的空字节加 `.pack` 字节模式读取 Pack 名称。
- Pack 名称按 UTF-8 解码并沿用扫描器的替换策略处理无效字节。
- 对结果进行文件名规范化、大小写不敏感去重并保留原始顺序。
- 返回前按集中定义的大小写不敏感名单过滤原版 Pack。
- 文件无法读取时抛出可由现有 RPC 错误包装处理的异常。
- 成功读取但没有匹配项时返回空列表。

原版 Pack 过滤名单集中定义，至少覆盖 WMM 已知核心 Pack 与参考实现的 WH3 条目；所有比较与启用流程复用同一名单，避免两种功能结果不一致。

### 后端 RPC

`backend/api.py` 注册只读 RPC `get_save_mods(save_name)`，返回：

```json
{
  "save": { "name": "campaign.save", "path": "...", "modified_at": 0, "size": 0 },
  "pack_names": ["first.pack", "second.pack"]
}
```

RPC 不接收当前启用 ID，也不修改播放集。

### 前端 Store

`frontend/src/store.js` 新增四个聚焦动作：

- `readSaveMods(saveName)`：调用 `get_save_mods` 并返回规范化结果。
- `enableModsFromSave(saveName)`：将 Pack 名称映射到 `store.mods`，完全替换 `activeIds`，设置 `dirty`，调用 `recordCurrentPlaysetChange()`，并返回已启用与缺失统计。
- `compareSaveMods(saveName)`：使用当前 `activeIds` 构造三组比较结果，不修改状态。
- `selectAllMods(orderedIds)`：校验 ID 属于当前 `modMap`，按传入顺序写入多选状态并加载首项预览。

同名 Pack 出现多个本机候选时，沿用加载清单导入的确定性选择规则：优先 Data 来源，其后按稳定 ID 排序。扫描器已经合并的 Data/Workshop 精确重复不重新展开。

### 存档列表与比较窗口

`frontend/src/components/SaveGamesModal.vue`：

- 每个存档行保留“载入”，并增加“启用存档中使用的模组”和“比较存档中的模组”。
- 新按钮分别发出 `enable-mods` 与 `compare-mods` 事件。
- 忙碌期间禁用会触发读取或写入的操作。

新增 `frontend/src/components/SaveModsComparisonModal.vue`：

- 独立显示三组比较结果及数量。
- 支持长列表滚动、空组提示和关闭操作。
- 不拥有或修改播放集状态。

`frontend/src/App.vue` 负责把存档窗口事件连接到 Store，并保存当前比较结果这一瞬时 UI 状态。

### 模组列表快捷键

`frontend/src/components/ModList.vue` 在列表边界内处理键盘事件：

- 仅响应 `event.ctrlKey && event.key.toLowerCase() === 'a'`。
- 编辑控件触发的事件直接放行。
- 有可见模组时阻止浏览器默认行为，并发出当前可见 ID 顺序。
- `App.vue` 将该事件连接到 `store.selectAllMods()`。

## 错误处理与反馈

- 路径无效、存档不存在和读取失败沿用统一 RPC 错误与 Store toast。
- 发生错误时不调用 `replaceActiveIds()`。
- 启用成功后提示启用数量；存在缺失 Pack 时使用警告级提示并包含缺失数量。
- 比较结果为空时仍打开窗口，并在三个分组中显示对应空状态。
- 新增按钮、标题、分组、空状态、忙碌文本和 toast 必须同时提供简体中文、英语、韩语、俄语和日语。

## 非目标

- 不自动订阅、下载或安装存档缺失模组。
- 不改变存档载入命令。
- 不改变搜索语法、排序语义或隐藏模组规则。
- 不增加跨两个列表的一次性全选。
- 不执行 0.6.0 正式发布流程。

- 比较窗口准确显示三组差异，不修改播放集。
- `Ctrl+A` 只选择当前列表当前可见的模组。
- 五种内置语言均无缺失新增文案。
