# WMM 0.6.5 Version and Changelog Design

## Scope

Advance the source version of Wyccc's Mod Manager from `0.6.0` to `0.6.5`,
add a concise five-language in-app changelog for the completed user-facing
features, and add the changelog-writing rule requested by the user to
`AGENTS.md`.

This is a source-only version advance. It does not build, publish, commit,
tag, or create a release.

## Version contract

The following active source surfaces move to `0.6.5`:

- `backend/constants.py`
- `pyproject.toml`
- `frontend/package.json`
- `frontend/src/store.js`
- `packaging/version_info.txt`
- `README.md`
- `README.en.md`
- the newest entry in `backend/changelog.py`

`packaging/update-manifest.json` remains at the actually published `0.6.0`
artifact. Its URL, size, and SHA-256 must not be changed until a real 0.6.5
executable exists.

## Changelog contract

The 0.6.5 changelog uses the five built-in languages and contains only
user-visible functions and fixes:

1. Automatic game-data patch validation and regeneration at game launch.
2. Integer 1-5 unit-scale control with optional lord and hero health scaling.
3. Faster opening of the Game Data Modification window.
4. More reliable Workshop author names without losing cached names.
5. Separate browser and Steam-client Workshop links.
6. Working Skip Intro coverage.

It does not mention fingerprints, hashes, HTTP concurrency, retry algorithms,
Steamworks binding details, DB field names, Pack-entry payloads, or other
implementation details.

## Repository rule

Add this rule to `AGENTS.md`:

> 更新日志必须简洁明了，只介绍用户可见的功能与修复，不描述实现细节。

## Validation

- Version synchronization tests distinguish source version `0.6.5` from the
  last published update manifest `0.6.0`.
- Changelog tests require the new release in every built-in language and keep
  the historical 0.6.0 notes intact.
- A repository-rule test protects the requested writing principle.
- Backend tests, frontend tests, build, Ruff, compile checks, and
  `git diff --check` run before completion.
